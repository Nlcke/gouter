goTo

  • goTo<N>(name, params, update?): null | State<T, keyof T>

  • Go to state using current stack navigator. First it searches for nearest state with same key generated from passed name and params. If that state exists it merges passed params, otherwise it creates new state with them. Then it applies passed update (if any) in the order: params -> stack -> index. If a route has shouldGoTo then it may skip navigation. Returns root state if it's changed or null otherwise.
